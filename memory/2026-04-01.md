## Notes

- The `FeatureFlagRefreshProvider` calls `update()` on mount + every 5 minutes. The call on mount only fires if `session?.user` is truthy (guarded by `if (!session?.user) return`).
- `setFeatureFlag` uses `deleteMany` (not `delete`) when disabling — safe if no override exists (no error thrown).
- Portal middleware reads flags from JWT (`session.user.enabledFeatures`) — not from DB on every request. Propagates within 5 min via refresh, or immediately when admin toggles (explicit `update()` call).
