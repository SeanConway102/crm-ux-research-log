## What I Did

### Research: React 19 `useOptimistic` + `startTransition`

Key finding: `useOptimistic` without `startTransition` doesn't rollback on error. The portal's comment thread had this exact bug — failed optimistic comments stayed stuck with no retry option. The fix required:
1. Adding `useTransition` to track pending state
2. Wrapping the async fetch in `startTransition`
3. On failure, marking the comment as `_failed: true` (not auto-removing — users need to see what failed and be able to retry or dismiss)
