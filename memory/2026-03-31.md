# 2026-03-31 — Client Portal Run

## Research: Vercel Flags SDK + Edge Config

Studied Vercel's `@vercel/flags` SDK and Edge Config for middleware-based feature flag enforcement.
**Key finding:** Our current Prisma-based approach is correct for page-level enforcement.
Middleware CANNOT use Prisma (Edge Runtime vs Node.js). The right pattern is:
- Page components (Node.js runtime) enforce flags via `isFeatureEnabled()` + redirect
- Middleware can't do this without Edge Config as a secondary store

Vercel Flags SDK would solve the middleware gap but adds infrastructure dependency. Decision: stick with page-level enforcement for now. Documented in `research-log.md` [arch].

Also researched TDD in Next.js RSC context — direct unit testing of Server Components is impractical; pure-function extraction + in-memory simulation is the right approach (already what we're doing).
