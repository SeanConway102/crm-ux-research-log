- **Research logged:** `memory/research-log.md` — document the stale JWT / flag propagation problem and fix decision.

**Build (Part 2):**
- Fixed the stale JWT problem:
  1. Reduced `session.maxAge` from default (30 days) to 3600 seconds (1 hour) in `lib/auth.ts`
  2. Created `hooks/useFeatureFlagRefresh.ts` — client hook that calls `session.update()` every 5 minutes, forcing NextAuth to re-call `jwt()` callback which re-fetches fresh flags from DB (via in-memory cache)
  3. Created `components/portal/FeatureFlagRefreshProvider.tsx` — client wrapper
  4. Added provider to `app/(portal)/layout.tsx`
  5. Added `__tests__/unit/lib/auth.test.ts` documenting the session config

**Result:** Feature flag changes now propagate within ~5 minutes for active users (instead of up to 30 days). 224 tests passing (added 2 new tests).
