# Curiosity — Find What You Don't Know

After completing a task or during a heartbeat cycle, run this check:

## Step 1: Scan for Triggers
Ask yourself:
- **Novelty:** Did I encounter something new or unexpected?
- **Complexity:** Is there a system I don't fully understand?
- **Uncertainty:** Are there outcomes I can't predict?
- **Conflict:** Does something contradict what I know?

## Step 2: Generate Questions at Three Depths
- **Surface (What?):** What is this? What changed? → Answer immediately if possible.
- **Mechanism (How?):** How does this work? How are these connected? → Search memory, then research if needed.
- **Strategic (Why/What if?):** Why is it this way? What would happen if it changed? → Log for deeper investigation.

## Step 3: Prioritize
Only pursue questions that:
1. Are relevant to your current project goals
2. Have actionable answers (you can DO something with the answer)
3. Are closable (not an infinite rabbit hole)

## Step 4: Act on Top Question
- If quick (<1 min): search memory, answer, move on
- If needs research: log to today's daily memory file for investigation
- If strategic: note it and revisit when more data accumulates

## Step 5: Record What You Learned
Write findings to `memory/YYYY-MM-DD.md`. Tag important discoveries for promotion to MEMORY.md.

## Anti-Patterns
- Don't chase interesting but irrelevant questions (stay goal-focused)
- Don't go down rabbit holes (timebox investigation to 5 minutes max)
- Don't skip this because you're "busy" — curiosity IS the work of getting better
