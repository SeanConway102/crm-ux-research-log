### Build: CRM Undo for Stage Moves
Implemented 5-second undo for drag-to-move in the CRM tickets queue (board view).

**File changed:** `crm-frontend/app/(dashboard)/tickets/page.tsx`

**What was added:**
- `undoTimersRef` + `undoMapRef` — track pending undo operations per ticketId
- Modified `handleDragEnd`: stores previousStageId, shows "Moved to [Stage] — Undo" toast with 5s window, cancels prior undo timer if ticket moved again
- New `handleUndoMove(ticketId, previousStageId, previousStageName)`: clears timer, reverts optimistically, calls `moveStage(previousStageId)`, shows confirmation toast

**UX flow:**
1. Agent drags ticket → optimistic update fires → "Moved to [Stage]" toast with "Undo" button (5s)
2. Agent clicks "Undo" within 5s → ticket reverts, "Moved back to [Stage]" toast confirms
3. Agent drags same ticket again before 5s → old undo cancelled, new undo registered
4. 5s passes without undo → timer clears, move is finalized
