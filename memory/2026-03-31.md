# 2026-03-31

## CT Website Co. Client Portal — Phase 1 Work

### What I did today

**Phase 1 Foundation** was mostly scaffolded previously. Today's work fixed three gaps:

1. **`extractSubdomain()` test fix** (`lib/tenant.ts`, `__tests__/unit/lib/tenant.test.ts`)
   - The subdomain extraction function was returning `null` for production-domain tests because `ROOT_DOMAIN` was set to `localhost:3000` in `.env`. Tests expected it to handle `portal.ctwebsiteco.com` and `admin.acme-corp.clientsite.com`.
   - Fix: Added optional `rootDomainOverride` parameter to `extractSubdomain()` so tests can pass the correct domain explicitly. Updated tests to use `rootDomainOverride` for production domain cases.
   - All 5 tenant unit tests now pass.

2. **NextAuth v5 Prisma schema** (`prisma/schema.prisma`)
   - The Prisma schema was missing the tables required by NextAuth v5 with PrismaAdapter: `Account`, `Session`, `VerificationToken`.
   - Added these tables + inverse relations on `PortalUser` (`accounts Account[]`, `Session Session[]`).
   - Ran `prisma format` to auto-fix relation field declarations.
   - Applied schema with `prisma db push`.

3. **Missing `/api/auth/onboarding` route** (`app/api/auth/onboarding/route.ts`)
   - The onboarding page POSTs to `/api/auth/onboarding` but the route didn't exist.
   - Created the route: validates password (Zod v4 API: `.object()` + `.issues[]`), hashes with bcrypt, updates `PortalUser.hasCompletedOnboarding = true`.
   - Also fixed Zod v4 API usage (was using `.errors` instead of `.issues`).

### Important dev note
**System has a global `DATABASE_URL` env var** pointing to a Neon PostgreSQL instance. This overrides the project's `.env` file when running Prisma CLI commands. Workaround: always `unset DATABASE_URL` before running `pnpm db:push`, `pnpm db:migrate`, etc.
